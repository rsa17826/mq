# MathQuest Item Randomizer Implementation Guide

## Overview
This item randomizer system creates randomized item spawn locations and hooks into the `setManagerData()` function to manage item collection logic. It follows the same pattern as the entrance randomizer but operates at the item level.

## Files Provided

### 1. `shuffle_items.py`
**Purpose:** Generates randomized item location data

**What it does:**
- Defines all collectible item types (food, gems, bombs, keys, etc.)
- Maps item instances to room locations with spatial coordinates
- Supports seeded randomization for reproducible runs
- Generates `json/items.json` with location data

**Usage:**
```bash
python shuffle_items.py [SEED]           # With seed
python shuffle_items.py 12345            # Example seed
python shuffle_items.py --no-shuffle     # Keep vanilla distribution
```

**Output:** `json/items.json`
- `seed`: The seed used for generation
- `totalItems`: Total item instances across all items
- `totalLocations`: Number of spawn locations
- `itemLocations`: Array of item location objects

**Item location object structure:**
```json
{
  "globalId": 0,
  "itemName": "apple",
  "instanceId": 5,
  "roomNorth": 20,
  "roomEast": 20,
  "spawnX": 50.5,
  "spawnY": 100.2,
  "xIsEven": 1,
  "yIsEven": 0
}
```

### 2. `patch_items.py`
**Purpose:** Generates and patches MathQuest.js with item randomization logic

**What it does:**
- Reads `json/items.json` generated by `shuffle_items.py`
- Creates efficient lookup maps for quick item access by room
- Hooks into `setManagerData()` function
- Injects telemetry/debug logging
- Prevents infinite loops with safety flags

**Usage:**
```bash
python patch_items.py
```

**Requirements:**
- Input: `MathQuest/MathQuest.base.js`
- Input: `json/items.json` (from shuffle_items.py)
- Output: `MathQuest/MathQuest.js` (patched version)

## Integration with Existing Code

### Key Integration Points

**1. setManagerData() Hook**
The patch wraps the existing `setManagerData()` function you've already added:

```javascript
function setManagerData(k, v, vv) {
  if (vv === undefined) {
    manager[k] = v
  } else {
    manager[k][v] = vv
  }
}
```

When called with item names, the patch detects and logs them:
```javascript
setManagerData("apple", manager.apple + 1)  // Detected as item gain
setManagerData("honey", manager.honey - 1)  // Detected as item loss
```

**2. itemTile and itemDisplay**
The existing visual display system remains unchanged:
- `itemTile[]` - Visual representation of items in the world
- `itemDisplay[]` - Text/UI display of player inventory
- These are NOT affected by the randomizer patch

**3. Manager Properties**
All item counts are stored on the manager object:
- `manager.apple`, `manager.honey`, `manager.grapes`, etc.
- `manager.bomb`, `manager.ruby`, `manager.aurastone`, etc.
- The patch reads/tracks these during gain/loss operations

## How It Works

### Flow Diagram
```
Player touches item in room (north, east)
         ↓
itemTile[x].hitTest() triggered
         ↓
setManagerData("itemName", value)  ← PATCH INTERCEPTS HERE
         ↓
IR_ROOM_MAP lookup: room → available items
         ↓
Find item in randomized pool
         ↓
Update manager.itemName + manager.itemDisplay
         ↓
itemTile visual updates
```

### Room Lookup
The patch builds a room-based map for O(1) lookups:
```javascript
IR_ROOM_MAP = Map {
  "20_20" → [
    { itemName: "apple", globalId: 0, spawnX: 50, spawnY: 100 },
    { itemName: "honey", globalId: 1, spawnX: 150, spawnY: 200 }
  ],
  "20_21" → [
    { itemName: "grapes", globalId: 5, ... }
  ]
}
```

### Debug Logging
All item operations log to console:
```javascript
[IR DEBUG] Item change detected: apple → 5
[IR DEBUG] Room has randomized items. Checking for matches...
[IR DEBUG] Found item 'apple' in room 20_20
[IR DEBUG] Applying item randomization
```

## Advanced Features You Can Add

### 1. **Item Pool Exhaustion**
Track when all copies of an item are collected and prevent duplicates:
```javascript
if (IR_ITEM_TRACKING.has(globalId)) {
  // Already collected, don't spawn again
  return;
}
IR_ITEM_TRACKING.set(globalId, true);
```

### 2. **Spatial Disambiguation**
For rooms with overlapping item spawn locations, use parity matching:
```javascript
var playerX = manager.char[0].get_x();
if (Math.abs(playerX - itemLoc.spawnX) < TOLERANCE && 
    (playerX % 2 === 0) === (itemLoc.xIsEven === 1)) {
  // Match found
}
```

### 3. **Item Pool Randomization**
Instead of fixed locations, pull from a shuffled pool:
```javascript
var availableItems = IR_ROOM_MAP.get(currentRoom);
var randomItem = availableItems[Math.floor(Math.random() * availableItems.length)];
setManagerData(randomItem.itemName, manager[randomItem.itemName] + 1);
```

### 4. **Cross-Room Item Gating**
Create logic where certain items only appear after others are collected:
```javascript
if (itemLoc.itemName === "key" && manager.apple < 10) {
  return; // Don't spawn key until 10 apples collected
}
```

## Combining with Entrance Randomizer

Both patches can run together:

1. Run entrance shuffle: `python shuffle_exits.py [SEED]`
2. Run item shuffle: `python shuffle_items.py [SEED]`
3. Run entrance patch: `python patch_game.py`
4. Run item patch: `python patch_items.py`
5. Load patched `MathQuest.js`

The two systems operate independently and won't conflict as they:
- Hook different manager operations (room transitions vs. item gains)
- Use separate lookup maps (ER_MAP vs. IR_ROOM_MAP)
- Have independent safety flags

## Common Issues & Solutions

**Issue:** "items.json not found"
- **Solution:** Run `shuffle_items.py` first to generate the JSON

**Issue:** "Patch anchor not found"
- **Solution:** Ensure you're patching `MathQuest.base.js`, not `MathQuest.js`

**Issue:** Items not updating in UI
- **Solution:** Verify `itemDisplay` array is being updated after setManagerData() call

**Issue:** Duplicate items spawning**
- **Solution:** Add item tracking to IR_ITEM_TRACKING map to prevent re-spawns
