# Item Randomizer: Setup & Implementation Guide

## Quick Start

### 1. Prerequisites
- Python 3.7+
- Your existing `MathQuest_base.js` with `setManagerData()` function already added
- Working entrance randomizer system (optional, but recommended)

### 2. File Structure
```
project/
├── MathQuest/
│   ├── MathQuest.base.js          (source - unmodified)
│   └── MathQuest.js               (generated - patched version)
├── json/
│   ├── connections.json           (entrance rando)
│   └── items.json                 (generated by shuffle_items.py)
├── shuffle_exits.py               (entrance rando generator)
├── patch_game.py                  (entrance rando patcher)
├── shuffle_items.py               (item rando generator) ← NEW
├── patch_items.py                 (basic item patcher) ← NEW
└── patch_items_advanced.py        (advanced item patcher) ← NEW
```

## Usage Workflows

### Workflow A: Item Rando Only (No Entrance Rando)

```bash
# Step 1: Generate item locations
python shuffle_items.py 12345

# Step 2: Patch the game
python patch_items.py
# or for advanced features:
python patch_items_advanced.py

# Step 3: Load MathQuest.js in your game
```

### Workflow B: Item Rando + Entrance Rando (Recommended)

```bash
# Step 1: Generate entrance locations
python shuffle_exits.py 12345

# Step 2: Generate item locations
python shuffle_items.py 12345

# Step 3: Patch entrances
python patch_game.py

# Step 4: Patch items
python patch_items_advanced.py

# Step 5: Load MathQuest.js in your game
# Both systems work together seamlessly
```

### Workflow C: Vanilla Distribution (Testing)

```bash
# Generate vanilla item locations (no shuffle)
python shuffle_items.py --no-shuffle

# Generate vanilla entrance locations (no shuffle)
python shuffle_exits.py --no-shuffle

# Patch normally
python patch_game.py
python patch_items_advanced.py
```

## Configuration

### shuffle_items.py Settings

Modify `ITEM_DEFINITIONS` to change item counts:
```python
ITEM_DEFINITIONS = {
    "apple": 15,         # Change how many apples exist
    "honey": 8,          # Change how many honeys exist
    "bomb": 20,          # etc...
}
```

Adjust `VALID_SPAWN_POSITIONS` to change where items can appear:
```python
VALID_SPAWN_POSITIONS = [
    (i * BLOCK_W + BLOCK_W/2, j * BLOCK_H + BLOCK_H/2)
    for i in range(14)           # 14 blocks wide
    for j in range(11)           # 11 blocks tall
    if not (i in [0, 13] or j in [0, 10])  # Exclude edges
]
```

Customize item distribution by room:
```python
if dist_from_center <= 2:
    items_in_room = random.randint(3, 6)  # Central rooms: 3-6 items
elif dist_from_center <= 5:
    items_in_room = random.randint(2, 4)  # Mid rings: 2-4 items
else:
    items_in_room = random.randint(1, 3)  # Edges: 1-3 items
```

### Choosing Between Patch Versions

**Use `patch_items.py` if you want:**
- Basic item tracking
- Minimal overhead
- Simple debug logging
- Foundation for custom extensions

**Use `patch_items_advanced.py` if you want:**
- Spatial disambiguation for overlapping items
- Collection tracking with debouncing
- Player proximity detection
- Real-time telemetry statistics
- Deadlock prevention with safety flags

## Output Format

### items.json Structure

```json
{
  "seed": 12345,
  "totalItems": 245,
  "totalLocations": 127,
  "itemLocations": [
    {
      "globalId": 0,
      "itemName": "apple",
      "instanceId": 0,
      "roomNorth": 20,
      "roomEast": 20,
      "spawnX": 50.5,
      "spawnY": 100.2,
      "xIsEven": 1,
      "yIsEven": 0
    }
  ]
}
```

**Field Meanings:**
- `globalId`: Unique ID for this item instance globally (0-245)
- `itemName`: Type of item (apple, honey, bomb, etc.)
- `instanceId`: Which copy of this item type (0-14 for apples)
- `roomNorth`: Room's north coordinate
- `roomEast`: Room's east coordinate
- `spawnX`: X pixel coordinate within room
- `spawnY`: Y pixel coordinate within room
- `xIsEven`: 1 if X coordinate is even, 0 if odd (for disambiguation)
- `yIsEven`: 1 if Y coordinate is even, 0 if odd (for disambiguation)

## Advanced Features

### Tracking Collected Items
The advanced patch maintains `IR_ITEM_COLLECT_LOG`:
```javascript
IR_ITEM_COLLECT_LOG = Map {
  0 → { itemName: "apple", collectedAt: 1234567890, room: "20_20", playerPos: {...} },
  1 → { itemName: "honey", collectedAt: 1234567895, room: "20_21", playerPos: {...} }
}
```

### Accessing from Console
```javascript
// See all collected items
console.log(window.IR_ITEM_COLLECT_LOG)

// Check specific room's items
console.log(window.IR_ROOM_ITEM_MAP.get("20_20"))

// Check if an item is collected
var isCollected = window.IR_ITEM_COLLECT_LOG.has(0)
```

### Custom Extensions

**Example: Item Limit**
Prevent collecting more than N of an item:
```javascript
if (isGain && manager[k] >= 10) {  // Max 10 apples
  console.log("[IR LIMIT] Cannot hold more " + k)
  return  // Don't apply the gain
}
```

**Example: Item Dependencies**
Only allow collecting certain items after prerequisites:
```javascript
if (k === "key" && manager.apple < 5) {
  console.log("[IR GATED] Need 5 apples before collecting keys")
  return
}
```

**Example: Random Item Pool**
Give a different item type from the room's pool:
```javascript
var roomItems = IR_ROOM_ITEM_MAP.get(playerPos.room)
var randomItem = roomItems[Math.floor(Math.random() * roomItems.length)]
setManagerData(randomItem.itemName, manager[randomItem.itemName] + 1)
```

## Troubleshooting

### Error: "items.json not found"
```
Solution: Run shuffle_items.py first
python shuffle_items.py 12345
```

### Error: "anchor not found" or "expected exactly one occurrence"
```
Solution: Make sure you're patching MathQuest.base.js, not MathQuest.js
SRC_PATH should point to: MathQuest/MathQuest.base.js
```

### Error: Syntax error in patched JavaScript
```
Solution: Check that all quotes and braces are balanced in the PATCH string
Verify: Python string formatting with {{ and }}
```

### Items not appearing in console
```
Solution: Check that items are actually being collected
Verify: manager.apple, manager.honey, etc. are changing
Note: Some items (gold, diamonds) may not be in IR_SAFE_ITEMS list
```

### "Infinite loop" or "Stack overflow"
```
Solution: The IR_UPDATING_INTERNAL flag prevents this
If it still occurs:
  - Check that setManagerData() isn't being called recursively
  - Verify IR_UPDATING_INTERNAL is properly set/cleared
  - Check for nested setManagerData() calls in your custom code
```

### Entrance rando and item rando conflict
```
Solution: They shouldn't conflict (they use separate hooks and maps)
Verify:
  - Both patches use their own safety flags (erUpdatingInternal, IR_UPDATING_INTERNAL)
  - Both use separate Map objects (ER_MAP vs IR_ROOM_ITEM_MAP)
  - Both hook different operations (room changes vs item gains)
```

## Performance Considerations

- **Memory:** ~50KB for 127 item locations (scales linearly)
- **Lookup Time:** O(1) room access, O(n) item search within room
- **Collection Overhead:** <1ms per item gain (with debouncing)
- **Console Logging:** ~500 bytes per debug message

**Optimization Tips:**
1. Limit room size or use quadtrees for very large item pools
2. Cache `irGetPlayerPos()` results
3. Reduce console logging for production
4. Batch room lookups if processing multiple items
